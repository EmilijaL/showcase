Unlike native JSF validation error messages, in a bean validation message by default the label of the component where a validation constraint violation originated from can not be displayed in the middle of a message.

Using the javax.faces.validator.BeanValidator.MESSAGE bundle key such label can be put in front or behind the message, but that's it.

With the JsfLabelMessageInterpolator a label can appear in the middle of a message, by using the special placeholder {jsf.label} in bean validation messages.
